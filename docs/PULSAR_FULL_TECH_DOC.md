**Название:** Ежедневный отчёт ООО Пульсар

### 3.6 РЕМОНТЫ (СЕРВИС)

**Доступ:** Слесарь + Директор

**Главное меню сервиса:**
- Открыть ремонт
- Активные ремонты
- Запчасти (подраздел)

**Алгоритм открытия ремонта:**
1. Машина: 803 | 008 | 709 | 137 | 555 | 143 | 729
2. Пробег (км) — числовое (вводится вручную, в будущем из Глонасс)
3. Категория неисправности: Двигатель | Ходовка | Электрика | Тормоза | Трансмиссия | Шины | Прочее
4. Описание поломки — текстовое поле
5. Кто ремонтирует:
   - **Свой ремонт** → карточка создаётся, сумма не запрашивается
   - **Сторонний сервис** → выбор: Агат | + Другое (ввод названия) → ввод суммы по акту → карточка создаётся

**Карточка ремонта содержит:**
- ID (формат: r[ДДММГГ][время])
- Дата открытия
- Машина, пробег, категория, описание
- Тип ремонта (свой / сторонний), если сторонний — название сервиса
- Статусы: Открыт → В работе → Ждёт запчасти → Закрыт
- Для своего ремонта: список работ, список расходов
- Итоговая сумма (для своего = сумма расходов, для стороннего = введённая при открытии)

**Закрытие ремонта:**
1. Нажать "Закрыть ремонт"
2. Ввести итоговый комментарий
3. Система фиксирует дату закрытия, ставит статус "Закрыт"

**Правила:**
- Сторонний сервис: сумма вводится один раз при создании, не редактируется
- Свой ремонт: сумма = детализация расходов
- Смешивать нельзя — либо внешний, либо внутренний

### 3.8 ДАШБОРД ДИРЕКТОРА

### 3.9 КАБИНЕТ ВОДИТЕЛЯ

**Переключатель:** Сегодня | Месяц

**Блок ЗП:**
- Заработано за период
- Авансы (минус)
- К выплате = Заработано − Авансы

**Логика расчёта ЗП водителей тонаров (Морозовский карьер):**
- Рейсы 1-3 за день: ₽2 000 за рейс
- 4-й рейс и более: ₽4 000 за рейс (бонус)
- Итого за 3 рейса: ₽6 000
- Итого за 4 рейса: ₽10 000
- Итого за 5 рейсов: ₽14 000

*Ставки для других карьеров и направлений прописываются отдельно (PENDING)*

**Система мотивации водителей:**
- Зарплата разделена 50/50: 50% за выполнение рейса, 50% за правильное оформление документа
- Т.е. при ставке ₽2000: ₽1000 за рейс + ₽1000 за правильно оформленный отчёт

**Статистика:**
- Рейсов сегодня / за месяц
- Тонн сегодня / за месяц

**План по объектам:**
- Прогресс-бар: перевезено / план (в тоннах)
- Список объектов с % выполнения

**Техобслуживание (ТО):**
- Счётчик км до замены масла
- В будущем: данные из Глонасс (Арус Нави API) — автоматически

**Авансы:**
- История авансов с датами и суммами
- Кнопка "Запросить аванс" → уведомление директору

- Вычеты = авансы + заправки

**Кнопка "Аванс":**
- Ввод суммы + комментарий
- Директору уведомление
- Сумма минусуется из ЗП

**Кнопка "Заправка":**
- Ввод суммы + АЗС
- Минусуется из ЗП автоматически

**Календарь выходов:**
- Визуальный календарь — зелёные дни когда работал

**История операций:**
- Выходы (зелёный +), авансы (оранжевый −), заправки (красный −)

### 3.11 ЭКОНОМИСТ — ПОДТВЕРЖДЕНИЕ НАЛИЧНЫХ

**Логика:** только для миксеров, сезонная (частники платят наличными)

**Флоу:**
1. Водитель в форме миксера указывает "Нал" и "У кого деньги"
2. Водитель нажимает "Передал деньги экономисту" → уведомление экономисту
3. Экономист видит карточку: водитель, сумма, способ (лично/перевод), время
4. Экономист нажимает "✓ Подтвердить получение"
5. Директор получает уведомление о двойном подтверждении

**Способы передачи:** лично в офисе | перевод на карту

### 6.1 Расчёт ЗП водителей тонаров (Морозовский карьер)
```
Рейс 1: ₽2 000
Рейс 2: ₽2 000
Рейс 3: ₽2 000
Рейс 4: ₽4 000 (БОНУС)
Рейс 5+: ₽4 000 каждый

Итого за N рейсов:
N=1: ₽2 000
N=2: ₽4 000
N=3: ₽6 000
N=4: ₽10 000
N=5: ₽14 000

Из которых:
- 50% за выполнение рейса
- 50% за правильно оформленный документ в системе
```
*Ставки для других карьеров (ВНК, Барзасский и др.) — PENDING, будут добавлены*

### 6.2 Расчёт ЗП слесарей
```
ЗП = Количество_смен × Ставка − Авансы − Заправки
Слесарь №1: ставка = ₽6 000/смена
Слесарь №2: ставка = ₽5 000/смена
Выплата: по запросу, обычно раз в неделю
```

### 6.3 Контроль наличных (миксеры)
```
1. Водитель завершает рейс с оплатой "Нал"
2. Указывает "У кого деньги": У водителя / У директора / Не забирал
3. При передаче → нажимает кнопку "Передал экономисту"
4. Экономист получает уведомление и подтверждает получение
5. Директор получает двойное уведомление (передал + получила)
```

### 6.4 Логика алертов директора
```
🔴 Красный (срочно):
- Водитель не внёс отчёт > 2 часов с начала рейса
- Машина в ремонте > 3 дней

🟡 Жёлтый (внимание):
- До ТО (замена масла) < 500 км
- Долг клиента просрочен > 3 дней
- Запрос аванса от сотрудника

🟢 Зелёный (информация):
- Новый рейс внесён
- Слесарь вышел на смену
```

### 6.5 Логика "ТН позже"
```
Если водитель не имеет ТН при отправке рейса:
1. Нажимает флаг "Нет ТН"
2. Рейс отправляется без фото
3. В таблице колонка "Нет ТН" = "да"
4. Позже: водитель открывает рейс, добавляет фото ТН
5. Директор/бухгалтер получают уведомление "ТН добавлена к рейсу от ДД.ММ"
```

### 6.6 Умная логика карьера ВНК
```
Если выбран карьер "ВНК (Березовка)":
→ Список заказчиков автоматически меняется на: [ВНК, + Другое]
→ При выборе заказчика ВНК: объекты = [ЗЖБИ, Кем ДСК, + Другое]
→ Работает для обоих материалов: Щебень и Песок
→ При смене карьера — заказчики возвращаются к стандартному списку
```

### 6.7 Голосовой ввод
```
Кнопка "Надиктовать рейс" → alert с примером: "Скажите: «803, щебень, 32 тонны»"
В финальной версии: интеграция с Telegram Voice API для распознавания
```

## БЫСТРЫЙ СТАРТ ДЛЯ НОВОГО РАЗРАБОТЧИКА

### Что нужно знать прежде всего:
1. Это SPA (Single Page App) — один HTML файл, экраны переключаются через JS
2. Роли определяются Telegram ботом, в прототипе — ручной выбор
3. Все данные идут в Google Sheets — нужно настроить API
4. Фото сохраняются через бот (Telegram file_id → Яндекс.Диск/Drive)
5. Уведомления директору — через бот notify_director()

### Файлы которые нужны для запуска:
```
index.html          ← основной Mini App (переименованный smart_fleet_industrial.html)
bot.py              ← Python Telegram бот
requirements.txt    ← pip зависимости
Procfile            ← для Railway
runtime.txt         ← python-3.10.13
.env                ← BOT_TOKEN, DIRECTOR_ID, WEBAPP_URL (не в git!)
```

### Порядок деплоя:
1. GitHub Pages → загрузить index.html
2. Railway → задеплоить бота из GitHub репо
3. Переменные в Railway: BOT_TOKEN, DIRECTOR_ID, WEBAPP_URL
4. В BotFather: настроить Menu Button → Web App URL
5. Директор назначает роли через /setrole
